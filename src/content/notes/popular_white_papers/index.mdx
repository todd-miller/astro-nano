---
title: "Key White-Papers to Read"
description: "A simple list of popular white-papers that describe core technologies"
date: "Dec 31 2024"
---

1. [MapReduce](https://static.googleusercontent.com/media/research.google.com/en//archive/mapreduce-osdi04.pdf): Simplified data processing on large clusters

2. [TAO](https://www.usenix.org/system/files/conference/atc13/atc13-bronson.pdf): Facebook’s distributed data store for the social graph

3. [Bitcoin](https://bitcoin.org/bitcoin.pdf): A Peer-to-Peer electronic cash system

4. [Google File System](https://static.googleusercontent.com/media/research.google.com/en//archive/gfs-sosp2003.pdf): A Scalable distributed file system

5. [BigTable](https://static.googleusercontent.com/media/research.google.com/en//archive/bigtable-osdi06.pdf): A Distributed storage system for structured data

6. [CAP Theorem](https://groups.csail.mit.edu/tds/papers/Gilbert/Brewer2.pdf): Perspectives on the CAP theorem

7. [Kafka](https://notes.stephenholiday.com/Kafka.pdf): A Distributed messaging system for log processing

8. [Chubby](https://static.googleusercontent.com/media/research.google.com/en//archive/chubby-osdi06.pdf): The lock service for loosely coupled distributed systems

9. [Log Structured Merge Tree](https://www.cs.umb.edu/~poneil/lsmtree.pdf): Data structure for efficiently storing key-value pairs in storage systems

10. [Google Spanner](https://static.googleusercontent.com/media/research.google.com/en//archive/spanner-osdi2012.pdf): Google’s globally distributed database

11. [Consistent Hashing](https://www.cs.princeton.edu/courses/archive/fall09/cos518/papers/chash.pdf): Distributed caching protocols for relieving hot spots on the World Wide Web

12. [Out of the Tar Pit](https://curtclifton.net/papers/MoseleyMarks06a.pdf): Functional programming to keep it simple

13. [The Part-Time Parliament](https://www.microsoft.com/en-us/research/publication/part-time-parliament/): Paxos consensus algorithm for distributed systems

14. [Raft](https://raft.github.io/raft.pdf): An understandable consensus algorithm

15. [TLA+](https://lamport.azurewebsites.net/tla/tla.html): High-level language for modeling concurrent and distributed systems

16. [Scaling Memcache at Facebook](https://www.usenix.org/system/files/conference/nsdi13/nsdi13-final170_update.pdf): Distributed cache architecture

17. [Borg](https://www.cs.cmu.edu/~harchol/Papers/EuroSys20.pdf): Google's cluster management system

18. [The Next 700 Programming Languages](https://www.cs.cmu.edu/~crary/819-f09/Landin66.pdf): A universal framework for language design

19. [Amazon Dynamo](https://www.allthingsdistributed.com/files/amazon-dynamo-sosp2007.pdf): Amazon’s highly available key-value store
